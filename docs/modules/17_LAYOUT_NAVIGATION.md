# Step 23 — Layout / Navigation Prompt

```text
Using the Father Business Management System context, build or update the responsive admin layout.

Goal:
The system should feel like an admin dashboard on desktop and a mobile app on phone.

Desktop layout:
- Sidebar navigation
- Top header
- Main content area
- Table-friendly layout

Mobile layout:
- App-style header
- Back button on sub-pages
- Bottom tab navigation for main modules
- Card-based list pages
- Sticky bottom action button when needed
- Safe-area padding for installed PWA
- No horizontal overflow

Main mobile bottom tabs:
- Dashboard
- Service Jobs
- Customers
- Products
- More

More menu should include:
- Services
- Price Catalog
- Models
- Reference Book
- Invoices
- Payments
- Expenses
- Reports
- Settings

Important:
- Do not use modal for create/edit forms
- Use separate pages
- Make buttons touch-friendly
- Make search easy to access
- Use Load More or infinite scroll for long mobile lists
- Support Khmer/English translation
- Support light/dark theme

Before coding:
1. Review existing layout files
2. List files to create/update
3. Then implement step by step
```
