# Step 3 — Mobile / PWA UI Rule Prompt

```text
Mobile Responsive & PWA UX Requirement:

Desktop should feel like a clean admin dashboard.
Mobile should feel like an installed PWA/mobile app.

Desktop:
- Use sidebar navigation
- Use top header
- Table layout is allowed
- Filters can be shown at the top
- Multi-column layout is allowed when suitable

Mobile:
- Use app-style header
- Add back button on create/edit/detail pages
- Use card list instead of wide table
- Use one-column forms
- Use large touch-friendly buttons
- Use sticky bottom primary action button where useful
- Use bottom navigation or sticky quick action where suitable
- Avoid hover-only interactions
- Avoid horizontal overflow
- Do not use modal for create/edit forms
- Use separate pages for create/edit/detail
- Use Load More or infinite scroll for long lists
- Support safe-area padding for installed PWA

Dialog rule:
Dialogs are allowed only for small actions such as:
- Delete confirmation
- Disable confirmation
- Cancel confirmation
- Status update
- Record payment
- Quick warning

Do not use dialog/modal for:
- Create customer
- Edit customer
- Create service job
- Create invoice
- Edit product
```
